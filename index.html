Elite Discoveries

A first-discovery tracker for Elite Dangerous that turns the game's raw journal logs into a personal exploration record.

Elite Dangerous writes everything a player does to local journal files, but the game itself has no screen where you can review which star systems you were the first commander to find. This app fills that gap: it reads the journal folder directly, follows which commander was logged in at the time of every scan, and applies the game's own WasDiscovered / WasMapped / WasFootfalled flags — so its results match the "Discovered by" tags shown in-game. No login, no API key, and nothing leaves the machine.

  • Parses 250+ journal files, keeps the earliest scan of each body, and skips scans made by other characters sharing the same save folder
  • Groups finds into the game's own categories — Earth-like worlds, water worlds, ammonia worlds, anomalies — with search, filters, sorting, and one-click name copy for the in-game galaxy map
  • "From here" mode reads your current location from the journals and re-sorts every discovery by light-year distance, banded nearest-first
  • Every planet gets a wireframe portrait drawn from its actual stats, alongside atmosphere, gravity, temperature, rings, bio/geo signals, and terraformable/landable badges
  • A Codex tab rebuilds your biological, geological, and astronomical finds from journal CodexEntry events, flagging first-logged entries
  • Optional commander profile card via Inara or EDSM keys, plus a working OAuth2 + PKCE login flow for Frontier's Companion API — keys stay server-side, never sent to the browser
  • Builds to a single self-contained .exe that opens in its own window, with installer and desktop-shortcut scripts

MW5 VKB HOSAS Mapper

A Windows GUI for setting up two VKB flight sticks in MechWarrior 5 — replacing hand-editing of an undocumented config-file format.

MechWarrior 5 has no menu for binding twin joysticks. Instead it reads a text file, HOTASMappings.Remap, that routes each physical input to one of two virtual devices the game understands — a format most players edit by hand, by trial and error. This app does the whole job visually: it detects both sticks over USB, shows exactly which axis or button you're moving in real time, builds the mapping, and writes a correctly formatted file straight into the game's folder with an automatic backup each time.

  • Live input panel identifies every control as you move it, with each button labeled by VKB's own grip names (Trigger, A1 Red Button, Coolie Hat…)
  • One-click HOSAS preset builds a sensible two-stick layout; a calibration wizard maps axes by watching which one you actually deflect
  • Binding editor is ordered like the game's own keybind menu — click an action's keybind, press a button on either stick, and it warns if that button is already in use
  • Photo-based stick diagram (standard or Omni-Throttle grip): press a physical control and its marker lights up green on the image
  • Auto-detects MechWarrior 5: Mercenaries, Clans, and MechWarrior Online — including MWO's different actionmaps.xml format, edited surgically so keyboard binds are left untouched
  • Auto-sync pushes changes to the connected game seconds after an edit, and can write in-game button commands to GameUserSettings.ini as well
  • Named profiles, per-axis invert/offset/deadzone tuning, and a warning banner when two identical sticks share a USB VID/PID (with the firmware fix explained)

Anime Tracker

A weekly airing-schedule tracker for the current anime season — one codebase shipped three ways: browser, installable desktop app, and native Android build.

Keeping up with a season of anime means checking several streaming services and release calendars; this app collapses that into one glanceable page. It queries the AniList and Jikan (MyAnimeList) APIs for the season's schedule, converts every air time into the viewer's local timezone, and lays out each day as a row of poster cards — what airs today, what episode number it's on, and where it streams. The whole thing is a single HTML file with no build step, which is also what makes the desktop and Android wrappers practical.

  • Live season data from the AniList and Jikan APIs, laid out on a Sunday-to-Saturday strip with back/forward week navigation
  • Each show card carries poster art, air time in your timezone, season and episode number, and SUB/DUB availability tags
  • Streaming-platform filter row — Crunchyroll, Netflix, Hulu, Prime Video, HIDIVE, and more — with matching badges on every card
  • A per-day count shows how many shows air that day; the current day is highlighted on the strip
  • The entire web app is one self-contained HTML file: open it in any browser and it just runs — no server, no install
  • Desktop edition installs Start-menu and desktop shortcuts with its own icon and opens in a standalone app window
  • Android edition is a native Gradle project that installs as a normal APK

Gunpla Tracker

A Gundam model-kit collection tracker organized by box art and grade — one codebase shipped as browser app, desktop app, and Android APK.

Collectors organize Gundam model kits (Gunpla) by the hobby's standard grades — EG, HG, RG, MG, and more. This app is a personal shelf: every kit displays as its box art in a grid grouped by grade with completion status (Backlog, Building, Built, Wishlist), search, and filters. All data stays on-device in IndexedDB with no account or server; works offline. Photos compress client-side. Kits without box-art photos get a generated grade-colored cover so the shelf never looks empty.

  • Box-art grid grouped by standard Gunpla grades (EG, HG, RG, MG, MGEX, FM, RE/100, PG, SD, MGSD, HiRM) with per-grade counts, search, and build-status filters
  • Detail page per kit showing description, specs badges, and a completed-build photo gallery with lightbox view
  • Instructions section — attach the kit's PDF manual and download it back anytime, plus built-in search of Bandai's free official manual library
  • Where-to-buy tabs by region (Japan, North America, Europe, Worldwide) that auto-generate store searches and link to your saved product links
  • Kits without a box-art photo get a generated grade-colored cover with kit name so the shelf stays visually consistent
  • Export/import moves the whole collection — photos and manuals included — between phone and desktop as one JSON file
  • Android edition adds a native file picker and saves manuals and exports to the phone's Downloads folder via JavaScript bridge

Plex Media Manager

A desktop app with a companion Android client that connects a Windows Plex host to a NAS, organizes a personal media library into Plex's exact naming conventions, and browses the result with cover art and search.

Plex only auto-fetches posters, summaries, and cast info when files follow its strict folder and naming rules, and a NAS-based setup adds its own pitfalls — Plex's background service can't even see mapped drive letters. This app manages that whole arrangement: it verifies the SMB connection to the NAS share, builds the folder tree Plex expects, and renames and relocates existing media files to match. Organizing is dry-run by default — every planned move is printed for review before anything is touched. One Python engine powers three surfaces: a windowed desktop app, an Android client, and a CLI for automation.

  • Connection check covers SMB reachability, a real write-permission test on the share, and a free-space report
  • Parses messy file names into Plex's Movie (Year) and Show — s01e01 structures; subtitle and metadata sidecar files are renamed and moved with their media
  • Dry-run by default: the organizer prints its full plan, and only --apply (or the desktop confirm) commits it
  • Uses UNC paths rather than mapped drives throughout, so the Plex service always sees the library
  • Point it at the Plex server (URL + token) and it becomes a library browser — cover art, categories, and name search across everything Plex has catalogued
  • The Android app pairs with the desktop app over home Wi-Fi by entering the address the desktop shows — no cloud account involved
  • Packaged as a standalone Windows .exe; all settings stay local in config.json
