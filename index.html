Elite Discoveries

A first-discovery tracker for Elite Dangerous that turns the game's raw journal logs into a personal exploration museum.

The app parses hundreds of the game's journal files locally to find every star system, planet, and moon the player was the first commander in the galaxy to discover, map, or land on. It runs as a zero-dependency Python server with a framework-free single-page UI, and packages into a single standalone Windows executable that opens in its own app window.

  • Parses 250+ journal files and attributes discoveries to the correct commander automatically
  • Categorized system browser — Earth-like worlds, water worlds, ammonia worlds, anomalies — with search, filters, and sorting
  • Procedurally generated holographic "portraits" for every planet, drawn from its real stats
  • Distance-from-here mode: type any system and every discovery re-sorts by light-year distance
  • In-game Codex reader, optional Inara/EDSM profile integration, and a full OAuth2 + PKCE flow for Frontier's Companion API
  • One-click build to a self-contained .exe with installer and desktop-shortcut scripts

MW5 VKB HOSAS Mapper

A full GUI for configuring dual VKB flight sticks in MechWarrior 5 — replacing hand-editing of an obscure config-file format.

MechWarrior 5 can't bind twin joysticks through its own menus; it reads a specially formatted HOTASMappings.Remap file. This app detects both physical sticks, shows live input feedback as you wiggle each axis and press each button, and writes a correctly formatted mapping file with automatic backups — then goes further and writes the in-game action bindings too.

  • Live device detection with real-time axis bars and button lights (pygame under the hood)
  • One-click HOSAS preset plus a guided calibration wizard — no guessing axis indices
  • Photo-based stick diagrams: press a physical button and its marker lights up on the image
  • Supports MechWarrior 5: Mercenaries, Clans, and MechWarrior Online — each with its own file format and action names
  • Auto-sync keeps the game's config current whenever a binding changes, with timestamped backups
  • Named profiles, per-axis invert/deadzone tuning, and a duplicate VID/PID conflict warning system

Anime Tracker

A weekly airing-schedule tracker for currently running anime — one codebase shipped three ways: browser, installable desktop app, and native Android build.

The tracker pulls live season data from public APIs and lays out each day's releases with poster art, episode numbers, air times in the local timezone, and sub/dub availability. A day strip and per-platform filters (Crunchyroll, Netflix, Hulu, Prime Video, and more) make it a glanceable "what's on today" dashboard.

  • Live schedule data from the AniList and Jikan (MyAnimeList) APIs
  • Week navigation with a per-day view, episode counts, and local air times
  • Filter by streaming platform, with per-show platform badges
  • Entire web app is a single self-contained HTML file — no build step, no server
  • Desktop edition installs as a pinnable taskbar app with its own icon and window
  • Android edition built as a native Gradle project

Plex Media Manager

A desktop app with a companion Android client that connects a Windows Plex host to a NAS, organizes a personal media library into Plex's exact naming conventions, and browses the result with cover art and search.

Plex only auto-fetches posters, summaries, and cast info when files follow its strict folder and naming rules. This app validates the SMB connection to the NAS, builds the expected library structure, and renames and relocates existing media files to match — safely, with a dry-run-by-default design. One Python engine powers three surfaces: a windowed desktop app, an Android APK that pairs with it over Wi-Fi, and a CLI for automation.

  • Desktop app in its own window, Android phone app, and a CLI — all sharing one engine
  • SMB reachability probe, write-permission test, and free-space report at a glance
  • Organizes messy file names into proper Movie (Year) and Show — s01e01 structures, dry-run by default with sidecar subtitle handling
  • Plex server integration: library browser with cover art, categories, and name search
  • UNC-path-first design so the Plex background service can always see the library
  • Packaged as a standalone Windows .exe; settings stay local in config.json
